# Pending Features Audit Plan

## Goal
Identify remaining feature gaps after reminder execution was added, and provide a prioritized implementation roadmap.

## Confirmed Pending Features
1. Legacy Node mode (`npm run start:legacy`) still does not execute intents; it only classifies and returns a limitation message.
2. Reminder recurrence is not implemented even though `frequency` is stored (`once` behavior only).
3. Natural-language time parsing is still limited to strict datetime formats; phrases like "tomorrow evening" are rejected.
4. Reminder management flows are missing: no user intents/endpoints for list, cancel, or update scheduled tasks.
5. Scheduler delivery hardening is incomplete: no retry/backoff policy, no max-attempts/dead-letter state, and no delivery idempotency token.
6. Callback security is minimal: scheduler->Node callback endpoint has no authentication/signature.
7. User timezone support is missing: reminders are interpreted/stored without per-user timezone preferences.
8. Documentation/config guidance is incomplete for scheduler/callback operational behavior and constraints.

## Recommended Implementation Order
1. Implement recurrence semantics (`daily`, `weekly`) with next-run computation after delivery.
2. Add task management intents (`list_tasks`, `cancel_task`) and DB query/update methods.
3. Improve time parsing using a deterministic parser + timezone-aware normalization.
4. Add retry policy (`attempt_count`, exponential backoff, terminal `failed` status).
5. Secure callback with shared secret header/HMAC validation.
6. Update README and env docs for scheduler settings and reminder behavior.

## Validation Plan
- Unit tests for recurrence and due-task transitions.
- Delivery integration tests for callback success/failure/retry paths.
- Intent-level tests for task listing/canceling and ambiguous time clarification.
- End-to-end bridge smoke test: schedule -> wait -> reminder delivered once.

## Done Criteria
- Reminders can be created, listed, canceled, and delivered reliably.
- Recurring reminders re-schedule correctly.
- Delivery path is authenticated and resilient to transient failures.
- User-facing docs accurately describe behavior and required env settings.
