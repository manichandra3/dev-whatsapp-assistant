# Incremental Implementation Plan (One by One)

## Goal
Close the remaining feature gaps in a safe, testable sequence, shipping one focused improvement at a time.

## Current Gaps To Implement
1. Legacy Node mode still returns raw intent JSON (no delivery execution).
2. `schedule_task` delivery lacks clarification behavior when metadata is incomplete.
3. `summarize_link` needs stronger fetch safeguards (content-type/size handling).
4. `execute_code` needs stricter sandbox/safety controls beyond timeout/output cap.
5. Delivery tests need completion and stronger assertions (including `general_chat`).

## Recommended Implementation Order

### Step 1 (Recommended): Complete and harden delivery tests first
- Finish `tests/test_delivery.py` incomplete cases.
- Add/expand tests for each intent handler success/failure behavior.
- Add targeted tests for timeout/output truncation and fallback text behavior.
- Keep existing tests passing.

### Step 2: Add `schedule_task` clarification behavior
- Detect missing schedule fields (`time`, `date`, `frequency`, etc.).
- Return a concise clarification response instead of blind persistence.
- Persist only when required schedule details are present.
- Add tests for both complete and incomplete metadata paths.

### Step 3: Harden `summarize_link` fetch path
- Validate URL scheme (`http`/`https` only).
- Enforce tighter timeout + maximum bytes read.
- Handle non-text content types with user-facing guidance.
- Add tests/mocks for timeout, bad URL, unsupported content type.

### Step 4: Harden `execute_code` sandbox constraints
- Keep Python-only execution, but tighten runtime and execution policy.
- Introduce stronger subprocess controls and explicit blocked patterns where feasible for MVP.
- Ensure deterministic, bounded outputs and robust logging.
- Add tests for blocked/unsafe patterns, timeout, runtime metadata, failure handling.

### Step 5: Bring Legacy Node mode to parity (or mark clearly)
- Preferred: mirror bridge behavior so `src/index.js` returns user-facing text rather than raw JSON.
- If parity is intentionally deferred, clearly gate and communicate mode limitations in runtime output/docs.
- Add/adjust tests or smoke verification path accordingly.

## Validation Per Step
- Run `npm run python:test` after each completed step.
- For Node legacy parity work, run `npm test` smoke command as configured.
- Confirm no regression in bridge mode response contract (`/message` returns plain text).

## Done Criteria
- All six intents produce plain-language outputs in bridge mode.
- Clarification and safeguards are enforced where metadata/network/code-exec risk exists.
- Tests for delivery/database pass with stronger coverage of failure paths.
- Legacy mode either achieves parity or is explicitly and consistently limited.
